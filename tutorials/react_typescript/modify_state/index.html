Modify State With Events¶

Prerequisites¶

Steps¶

We want to build a counter component which keeps track of clicks.

  1. Start writing the HTML inline:

    <div onClick={() => alert('Clicked!')}>
   <span>10</span>
</div>

  2. Click in browser

  3. Highlight and cut, then:

    class Counter extends React.Component {
  render() {
      return (

  4. Show it working in the browser.

  5. Add a prop for a label to show the count, make it optional with ? and put || ‘Count: ‘ in the span

  6. Extract to an interface for props

  7. Make an interface for CounterState and add to generic

  8. Make a constructor to setup the state

  9. Replace the <span> value with this.state.count

  10. Replace the onClick handler with:

    <div onClick={() => this.setState({count: this.state.count + 1})}>

  11. Why is the arrow function needed? Try without it.

  12. Use refactor to make the method:

    private increment() {
    this.setState({count: this.state.count + 1});
}

  13. Use a prop as the starting value….add starter?: number to the props

  14. Set initial state with count: this.props.starter || 0

  15. Show autocomplete on <Counter>

  16. Show mistakenly doing <Counter starter=”10”/>

  17. Shift click to increment by 10

  18. onClick={(event) => this.increment(event)}

  19. increment(event: React.MouseEvent<HTMLElement>)

  20. Mistake for const inc = 10 ? event.shiftKey : 1;

  21. Partial correction: const inc: number = 10 ? event.shiftKey : 1;

  22. Full correction: const inc: number = event.shiftKey ? 10 : 1;

  23. Make the interface less browser-y by moving the event unpacking to the onClick handler, thus passing only event.shiftKey:

    <div onClick={(event: React.MouseEvent<HTMLElement>) => this.increment(event.shiftKey)}>

  24. Even simpler, spread operator:

    <div onClick={({shiftKey}) => this.increment(shiftKey)}>

  25. Extract to a separate file

  26. Write a test…requires changing private
