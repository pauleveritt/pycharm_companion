Unit Testing with Jest and Enzyme¶

Let’s take a look at Jest testing in the context of a cra-ts project. Meaning, testing React components (stateful, stateless) that are written in TypeScript. This lesson doesn’t cover everything, just enough to get started.

  • Awesome that it just figures everything out in the config, even ghosting the suggestion for the two args

Prerequisites¶

  • Jest
  • Project Setup

Steps¶

  1. Make a Jest run config with the args

  2. Run tests, single, etc.

  3. Add to the code:

    const a = 1;
const b = 2;
expect(a).toBe(b);

  4. Reformat code, watch the re-run, show red colored gutter icon

  5. I wanted e (expected) instead of b, use Refactor -> Rename.

  6. Fix failure, show test passes.

  7. Jump between test and source with Shift-Command-T

  8. Terminal

  9. Install Enzyme npm install -D enzyme enzyme-adapter-react-16 react-addons-test-utils @types/enzyme @types/enzyme-adapter-react-16

  10. Add a bootstrap file for Enzyme at src/setupTests.js:

    import * as Enzyme from 'enzyme'
import * as Adapter from 'enzyme-adapter-react-16'

Enzyme.configure({
  adapter: new Adapter(),
})

  11. Add a test:

    it('renders the provided text', () => {
    const wrapper = shallow(<App/>);
    expect(wrapper.find('div').text()).toBe('Hello React');
});

  12. Let the IDE generate the shallow import, but with missing spacing

  13. Preferences, search for ES6 import braces (Code Style -> TypeScript -> Spaces -> Within -> ES6 import/export braces)

  14. TDD mode step one: source and test side-by-side mode
