Project Setup¶

Like all other frontend technologies, it’s difficult to get React, TypeScript, the tools, and the entire build environment configured in a new project. Once you do, everything changes and you have to start over.

Many frameworks have recently built CLI (command-line interpreter) tools that make those decisions for you, using best practices. They’ll generate a project with the right choices then update those choices as things evolve.

React’s create-react-app takes care of this. And it’s extensible with scripts such as react-scripts-ts which provides a TypeScript version of the generated project.

In this tutorial step, we use the IDE to generate the project, then look at the results.

Creation¶

First we create a new React project.

  1. Make sure NodeJS and create-react-app are installed on your system.
  2. Open PyCharm Professional
  3. In the New Project dialog, select React App as the project type.
  4. Set Location set to the directory you’d like the project to be generated in.
  5. Expand More Settings.
  6. Set Scripts version: to react-scripts-ts
  7. Click Create.
  8. In the next dialog, select Open in new window and click Ok.
  9. Wait a few minutes while create-react-app downloads a bunch of npm dependencies and generates the project.
  10. Close the Run tool window and expand the root folder.
  11. Put things in a local Git repository by selecting the menu VCS -> Enable VCS Integration then selecting Git.
  12. Select the folder/file entries for public, src, and everything in the root except .gitignore, then Ctrl-V and Add to VCS.
  13. Ctrl-V | Commit to commit the starting point.

Running¶

Let’s run some React stuff.

  1. Click the npm tool window (or open package.json and look at the run icons for scripts).
  2. Double-click start and look in the opened browser window.
  3. Open src/App.tsx, save a small change, and see the update in browser.
  4. In the npm tool window, double-click build, read output window for bundle size, right-click the new build directory, and Mark Directory As | Excluded.
  5. In tool window, double-click test
  6. Open src/App.test.tsx, make a small change adding expect(1).toBe(2); to the last line of the test, ignore the indentation warning, save.
  7. See the update.

What Happened¶

Before anything can happen, you have to install NodeJS on your system. Then, you have to install create-react-app either “globally” or somewhere that you can point the IDE to.

PyCharm Professional integrates with create-react-app to generate the new project. The IDE lets you pass an argument to the generator for the extra script to use. We want to use react-scripts-ts, a mature “fork” of create-react-app which changes the output to be React+TypeScript.

If yarn is installed, create-react-app will choose it as the package manager and generate a yarn-oriented project. PyCharm Professional will respect that decision.

It takes a while for create-react-app to download all the dependencies then generate the file. This will be faster in some subsequent runs, as the npm/yarn cache has many of the versions. Also, the IDE needs to index all the downloaded software which can also take a number of seconds.

The node_modules directory is detected as a library root and marked as “Excluded”. This speeds up indexing by not having those files reindexed as your code changes.
